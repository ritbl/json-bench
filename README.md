# JSON benchmark: `sonic-rs` vs DSL-JSON

This repository benchmarks serialization and deserialization of the same large JSON payload in:

- Rust with `sonic-rs`
- Java with DSL-JSON

The payload is generated by a script so both runtimes work from identical input.

## Layout

- `scripts/generate_big_json.py`: Python generator used inside the container
- `scripts/generate_big_json.sh`: Docker wrapper for dataset generation
- `rust-sonic/`: Rust benchmark using `sonic-rs`
- `java-dsljson/`: Gradle-based Java benchmark using DSL-JSON
- `scripts/run_rust.sh`: builds and runs the Rust benchmark
- `scripts/run_java.sh`: runs the Gradle Java benchmark
- `scripts/run_all.sh`: convenience wrapper

## 1. Generate a large JSON file

Generate a moderately large dataset:

```bash
./scripts/generate_big_json.sh --output data/big.json
```

Generate a larger one:

```bash
./scripts/generate_big_json.sh \
  --output data/big.json \
  --items 20000 \
  --variants 6 \
  --reviews 10
```

The generator is deterministic by default, so repeated runs with the same arguments produce the same payload.
The generator runs in Docker and bind-mounts the repository into `/workspace`.
Docker must be available locally for generation and for `run_all.sh` when the JSON file is missing.

## 2. Run the Rust benchmark

```bash
./scripts/run_rust.sh data/big.json
```

Optional:

```bash
./scripts/run_rust.sh data/big.json 15 5
```

Arguments:

- `path`
- `iterations` default: `10`
- `warmup` default: `3`

## 3. Run the Java benchmark

```bash
./scripts/run_java.sh data/big.json
```

Optional:

```bash
JAVA_RELEASE=25 ./scripts/run_java.sh data/big.json 15 5
```

Arguments:

- `path`
- `iterations` default: `50`
- `warmup` default: `10`

Notes:

- The Java source is intended for Java 25, but this machine currently has Java 21 installed. Use `JAVA_RELEASE=21` locally unless Java 25 is installed.
- The Java benchmark is a Gradle project and uses DSL-JSON annotation processing for compile-time serializers.

## 4. Run everything

```bash
./scripts/run_all.sh data/big.json
```

If the file does not exist, `run_all.sh` generates it first.

## Benchmark output

Both programs print:

- input size
- warmup and measured iteration counts
- deserialize timing summary
- serialize timing summary
- estimated throughput in MiB/s

## Fairness notes

- Both implementations deserialize the same generated payload into typed models.
- Both implementations serialize the typed in-memory model back to JSON bytes.
- The benchmark is a simple process-level harness, not JMH (java) or Criterion (rust).
- Use multiple runs and a quiet machine if you want cleaner comparisons.
